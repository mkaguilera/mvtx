Detailed Description

Instantiation of LockManager using AVL trees. AVL trees are capable of providing every operation in O(log(total_nr_intervals) + involved_nr_intervals). As a result, AVL trees are very efficient and natural choice for supporting the LockManager API.

Roll backwards from a specific time and lock everything until you encounter a frozen lock or the beginning of time. In case, it is impossible to proceed because of another regular (not frozen) lock, register to be resolved when this lock is frozen or released.

Parameters
key- Key for which lock is performed.
tid- Transaction ID for the transaction that performs this lock.
ts_end- Time for which lock procedure is initiated (inclusive).
is_read- Whether or not this is a read lock.
event- Event to run when backwards lock finishes.
ts_start- The earliest time that was actually locked (inclusive).
Returns
- Whether or not locking stopped at a frozen lock or the beginning of time.
